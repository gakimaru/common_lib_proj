【開発環境サンプル】

■common_lib_proj

ブランチ：proj_c

--------------------------------------------------------------------------------
▼内容

・共通ライブラリ common_lib と 機密性共通ライブラリ common_lib_confidential の
  開発プロジェクト用リポジトリ。

・各プロジェクト向けに提供するライブラリファイル(*.a/*.lib)を作成する。

・また、共通ライブラリの動作テストに対応し、テスト専用のプロジェクトと
  ソースファイル(*.h/*.inl/*.cpp)を扱う。

--------------------------------------------------------------------------------
▼ブランチ

・本流（master） ... 共通ライブラリ開発用
・（各プロジェクト向けブランチ）

※各プロジェクト向けブランチを使用するのは、各プロジェクト向けの
  ライブラリファイル(*.a/*.lib)を作成する時。

※ライブラリの動作テストプロジェクトは基本的に本流で扱う。
  各プラットフォーム向けのテストや設定に応じた挙動のテストには、
  ブランチを分けずに、多数のプロジェクトやビルドターゲットで分けて対応する。

--------------------------------------------------------------------------------
▼依存リポジトリ

・common_lib_individual       ... 共通ライブラリの動作設定（ヘッダーファイル）。
・common_lib                  ... 共通ライブラリ（ヘッダーファイル）。
・common_lib_src              ... 共通ライブラリ（ソースファイル）。
・common_lib_confidential     ... 機密性共通ライブラリ（ヘッダーファイル）。
・common_lib_confidential_src ... 機密性共通ライブラリ（ソースファイル）。

※common_lib_individual の本流には、例外的に多数のファイルを持つ。
  （通常は各プロジェクトで一ファイルだけ使用する）
  各種テストの為の設定ファイルを多数扱う。

--------------------------------------------------------------------------------
▼サブモジュール適用フォルダ

・/lib/individual/       ← common_lib_individual
・/lib/common/           ← common_lib
・/lib/common_src/       ← common_lib_src
・/lib/confidential/     ← common_lib
・/lib/confidential_src/ ← common_lib_src
